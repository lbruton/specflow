The problem with AI-assisted development

Every session starts from zero. Three failure modes compound over time.

🧊

Cold starts

The agent has no memory of yesterday. It doesn't know what it built, what broke, or what conventions you agreed on. You re-explain everything, every time.

💥

Context flooding

You stuff everything into CLAUDE.md — architecture, conventions, decisions, infrastructure. The file balloons. Token waste goes up. Attention quality goes down.

📉

No lifecycle

Agents jump straight to code. No requirements phase. No design review. No approval gates. The result: rework, scope creep, and features that don't match intent.

The SpecFlow ecosystem

SpecFlow orchestrates how work gets done. Five companion MCPs give AI agents persistent memory, semantic code understanding, and structured knowledge — replacing the bloated CLAUDE.md + ad-hoc prompting pattern.

SpecFlow — Orchestrator

Spec-driven lifecycle

Requirements → Design → Tasks → Implementation with dashboard approvals at every gate. Works with Claude Code, Gemini CLI, and Codex CLI — any MCP-compatible agent.

SessionFlow — Session Memory

Full session recall in seconds

Every conversation turn is indexed in real-time into a local vector database. Semantic search across all past sessions with date filtering — the agent remembers exactly where you left off. Forked from mwgreen/claude-code-session-rag.

DocVault — Knowledge Base

Cross-project ground truth

One Obsidian vault serves 8+ repos. Architecture, infrastructure, decisions, issues — all in structured markdown with graph visualization and wikilinks. Wins all tie-breaking conflicts.

ContextFlow — Code Intelligence

Semantic + structural code search

Self-hosted Milvus vector database indexes every codebase. Search code by meaning, not just keywords. Planned: merging structural graph analysis (callers, imports, dead code) into one unified MCP server. Forked from zilliztech/claude-context.

MemFlow — Episodic Memory

Curated cross-session memory

Retro lessons, handoffs, and session summaries stored as semantic embeddings. Currently powered by mem0 cloud API. MemFlow will be a self-hosted fork — zero cloud dependencies for the full stack.

Skill System — 6 shipped skills

Procedural knowledge routing

CLAUDE.md stays tiny — a routing table to skills. Ships with 6 lifecycle skills and 11 commands. Designed to compose with your own library — the author runs ~70 personal skills on top of these defaults.

Four-layer contextual memory

Not a hierarchy — four parallel systems, each authoritative in its own domain. Which layer holds the truth depends on what you’re asking.

MemFlow

MemFlow — Curated episodic memory

Retro lessons, cross-session decisions, handoffs, and session summaries stored as semantic embeddings. Currently powered by mem0 cloud API; self-hosted fork planned.

▸ Best for: past decisions, recurring patterns, “what did we agree on?”

SessionFlow

SessionFlow — Verbatim session history

Every conversation turn indexed in real-time into Milvus Lite. Semantic search with date filtering across all past sessions. Primary context source for /start and /prime.

▸ Best for: debugging history, “how did we fix that bug last month?”

DocVault

DocVault — Living documentation

Human-curated Obsidian vault. Architecture, infrastructure topology, data models, coding standards, deployment procedures. Version-controlled with graph visualization and wikilinks. Authoritative for anything documented — but subject to drift.

▸ Best for: code standards, architecture, infrastructure, “how should this work?”

Files

File Memory — Project context

CLAUDE.md, MEMORY.md, and project-scoped markdown at ~/.claude/projects/*/memory/. Agent-curated, human-reviewable. Loaded at session start. Stores user preferences, feedback, and external references.

▸ Best for: user preferences, project config, “how does this user like to work?”

▼ no fixed hierarchy — context determines which layer is authoritative

ContextFlow — code intelligence, self-hosted

Agents shouldn’t grep blindly through your codebase. ContextFlow combines semantic search (meaning-based) and structural analysis (callers, imports, dead code) into one self-hosted code intelligence layer. Both backends run on your infrastructure — no cloud dependencies.

Semantic Search — Milvus

Search by meaning

Ask “find the authentication middleware” and get results even if the code never uses the word “auth.” Vector embeddings capture semantic relationships across your entire codebase.

Persistent Indices

Index once, search forever

Code indices persist across sessions in Milvus. No re-indexing on every cold start. Incremental updates catch only what changed since the last index run. Unlimited project indices with full data sovereignty.

Hardened Fork

Production-stable

Forked from Zilliz’s MCP server with critical fixes: 30s fetch timeouts, gRPC connection guards, and pinned npm versions. No silent upstream breakage.

Structural Graph — Neo4j

Callers, imports, dead code

“What calls this function?” “Show me everything that imports this module.” “Find unused exports.” Structural queries that semantic search can’t answer. Cross-language support.

Complexity Analysis

Find hotspots before they burn

Cyclomatic complexity scoring, most-complex-function ranking, and call chain tracing. Identify the code that’s hardest to maintain before it becomes a problem.

Roadmap — Unified MCP

One server, both engines

Structural graph analysis is currently provided by Code Graph Context (separate MCP server). Planned: merge into ContextFlow — one MCP, one install, semantic + structural in a single query.

Every session learns from the last

Most AI tools treat each session as isolated. SpecFlow creates a continuous learning loop: sessions end with knowledge extraction, and new sessions start pre-loaded with everything that matters.

The continuous context lifecycle
┌─────────────────────────────────────────────────────────┐ │ THE CONTINUOUS LEARNING LOOP │ └─────────────────────────────────────────────────────────┘ Quick resume — /start (same-day, short breaks) Queries SessionFlow for recent conversation turns Pulls curated memories from MemFlow Checks git state + 5 most recent open issues Under 15 seconds, no indexing Full boot — /prime (after days away, new projects) Queries SessionFlow + MemFlow for full context Indexes codebase via ContextFlow Checks open issues + git status + code health Presents: “Here’s where you left off, here’s what’s next” During the day — sessions compound SessionFlow indexes every turn in real-time Work produces commits, specs, implementation logs /handoff relays state between terminal sessions End of session — /wrap (retro folded in) /wrap Step 1: Cleanup (stale branches, uncommitted work) Step 2: Retro — extracts prescriptive lessons Saves to MemFlow as structured retro-learning memories Step 3: Curated session summary → MemFlow SessionFlow has the verbatim turns; MemFlow gets the digest On-demand — /audit checks project health /audit Code quality + security scan Documentation drift detection Issue staleness check Actionable remediation report ┌──────────────────────────────────────────────────────────────┐ │ /start ──→ work ──→ /audit ──→ /wrap ──→ /start ──→ .. │ │ SessionFlow + MemFlow carry context across every session │ └──────────────────────────────────────────────────────────────┘
/start

Quick resume

Queries SessionFlow + MemFlow for recent context, checks git state and open issues. Under 15 seconds, no indexing. Use for same-day work or short breaks.

/prime

Full session boot

Queries SessionFlow + MemFlow, indexes codebase via ContextFlow, checks code health, gathers project status. Use after days away or on new projects.

/wrap

End-of-session orchestrator

One command to close a session cleanly: cleanup stale branches, run retrospective inline, save lessons + curated summary to MemFlow. SessionFlow indexes turns automatically — /wrap captures the curated layer.

/audit

On-demand project health check

Scans code quality, security posture, documentation drift, and issue staleness. Produces an actionable remediation report. Run anytime — before a release, after a sprint, or when things feel off.

/retro

Prescriptive lesson extraction

Not “what did we do” but “what should we do differently.” Extracts actionable rules from the conversation and saves them as structured memories. Future sessions apply these lessons automatically.

Parallel subagent dispatch

The biggest architectural difference from other spec workflows: tasks don't execute sequentially in one session. Each task runs through an isolated three-stage pipeline, and tasks with zero file overlap execute concurrently.

Subagent pipeline — per task
# Each task runs through 3 isolated stages Orchestrator reads task from spec → Implementer Agent fresh context, writes code, tests, commits → Compliance Reviewer reads actual code vs requirements → Quality Reviewer architecture, error handling, production readiness → log-implementation record artifacts → mark complete # File Touch Map identifies zero-overlap tasks # Those tasks dispatch simultaneously in batches Batch 1: Task 1 + Task 3 + Task 5 parallel Batch 2: Task 2 + Task 4 parallel Batch 3: Task 6 (depends on Task 2) sequential
